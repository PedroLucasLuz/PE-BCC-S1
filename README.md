# PROJETO DE EXTENSÃO

## 1. APRESENTAÇÃO

A lógica de programação é a base para o desenvolvimento de sistemas, aplicativos e soluções tecnológicas. Entretanto, muitos estudantes encontram dificuldades na compreensão dos conceitos fundamentais por considerá-los abstratos ou complexos.

Pensando nisso, este projeto de extensão propõe que os alunos se tornem multiplicadores do conhecimento, estudando e ensinando conceitos básicos de lógica de programação por meio da produção de conteúdos educativos e da realização de uma live aberta ao público.

---

## 2. OBJETIVO GERAL

Compreender e explicar conceitos fundamentais da lógica de programação de forma simples, prática e acessível para iniciantes.

---

## 3. OBJETIVOS ESPECÍFICOS

* Desenvolver o raciocínio lógico e computacional;
* Compreender os principais conceitos da lógica de programação;
* Aprimorar habilidades de comunicação e trabalho em equipe;
* Produzir materiais educativos para iniciantes;
* Aplicar conhecimentos de programação em situações práticas.

---

## 4. DESCRIÇÃO DO PROJETO

Os alunos serão organizados em grupos. Cada grupo ficará responsável por estudar um tema específico da lógica de programação e transformá-lo em uma mini aula para iniciantes.

A apresentação deverá conter:

* Explicação do conceito;
* Exemplo do cotidiano;
* Analogias que facilitem o entendimento;
* Exercícios simples;
* Demonstração prática em código;
* Participação de todos os integrantes.

Ao final do projeto, todos os grupos participarão de uma live educativa intitulada:

### **DESMISTIFICANDO A PROGRAMAÇÃO: APRENDENDO LÓGICA DE FORMA SIMPLES**

---

## 5. TEMAS DOS GRUPOS

### Grupo 1 – Variáveis

Conteúdos sugeridos:

* O que são variáveis;
* Armazenamento de informações;
* Tipos de dados básicos;
* Exemplos práticos.

### Grupo 2 – Operadores

Conteúdos sugeridos:

* Operadores aritméticos;
* Operadores relacionais;
* Operadores lógicos;
* Aplicações práticas.

### Grupo 3 – Estruturas Condicionais (IF/ELSE)

Conteúdos sugeridos:

* Tomada de decisão;
* Condições verdadeiras e falsas;
* Exemplos de aprovação/reprovação;
* Aplicações do dia a dia.

### Grupo 4 – Estruturas de Repetição (Loops)

Conteúdos sugeridos:

* Repetição de tarefas;
* Contadores;
* Laços de repetição;
* Exemplos práticos.

### Grupo 5 – Funções

Conteúdos sugeridos:

* Reutilização de código;
* Organização de programas;
* Parâmetros e retorno;
* Exemplos simples.

### Grupo 6 – Listas

Conteúdos sugeridos:

* O que são listas (vetores);
* Armazenamento de múltiplos valores;
* Acesso aos elementos por índice;
* Percorrendo listas com estruturas de repetição;
* Aplicações práticas no desenvolvimento de programas.

---

## 6. PRODUTO FINAL

Cada grupo deverá apresentar uma mini aula durante a live, contendo:

1. Apresentação do tema;
2. Explicação conceitual;
3. Exemplo do cotidiano;
4. Exercício resolvido;
5. Demonstração prática em código;
6. Conclusão.

Tempo sugerido por grupo: 10 a 15 minutos.

---

## 11. UTILIZAÇÃO DO GITHUB

Durante o desenvolvimento do projeto, cada grupo deverá utilizar o GitHub como ferramenta de versionamento e colaboração.

## Requisitos

Cada grupo deverá:

* Criar um repositório no GitHub;
* Adicionar todos os integrantes como colaboradores;
* Registrar as atividades por meio de commits;
* Manter os materiais organizados conforme a estrutura definida;
* Disponibilizar o link do repositório juntamente com a apresentação final.

---

---

## 13. PADRÃO DE COMMITS

Os commits deverão possuir mensagens claras e objetivas.

Exemplos:

```text
docs: adiciona roteiro da apresentação

feat: cria exemplo de estrutura condicional

feat: adiciona exercício sobre loops

fix: corrige erro no algoritmo de média

docs: atualiza README do projeto
```

Evite mensagens genéricas como:

```text
atualização

teste

mudanças

arquivo novo
```

---

## 14. ENTREGA FINAL

Cada grupo deverá entregar:

* Link do repositório GitHub;
* Slides da apresentação;
* Exercícios produzidos;
* Exemplos de código;
* Relatório final;
* Link da gravação da live.

A organização do repositório, a qualidade dos commits e a participação dos integrantes poderão ser consideradas na avaliação final do projeto.
