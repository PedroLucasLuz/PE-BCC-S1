Fala, pessoal! Aqui está uma lista de exercícios para ajudar a fixar a matéria. A ideia é testar o que vocês entenderam sobre funções na lógica de programação, do básico ao mais avançado.


**Como responder:** Nas questões de múltipla escolha, é só colocar um `x` dentro dos colchetes `[ ]`. Nas questões abertas, basta escrever sua resposta logo abaixo da pergunta.


---


### Nível Fácil


**1. De forma geral, o que é uma função na programação?**
- [ ] A) Uma variável que guarda números inteiros.
- [ ] B) Um tipo de erro que acontece no código.
- [ ] C) Um bloco de código reutilizável que faz uma tarefa específica.
- [ ] D) Uma estrutura que serve só para criar laços infinitos.


**2. Qual é a maior vantagem de usar funções no seu código?**
- [ ] A) Fazer o código rodar mais devagar para não travar o computador.
- [ ] B) Evitar ficar repetindo código, deixando tudo mais fácil de organizar e consertar depois.
- [ ] C) Acabar com a necessidade de usar outras variáveis.
- [ ] D) Fazer o programa rodar sem usar internet.


**3. O texto diz que uma função é parecida com uma "máquina de fábrica". Explique com as suas palavras por que isso faz sentido.**


**4. Na estrutura de uma função, o que é o "Parâmetro"?**
- [ ] A) É o resultado que a função devolve no final.
- [ ] B) É o nome da função que você chama.
- [ ] C) É o bloco de código que ela vai rodar.
- [ ] D) É uma variável (ou dado) que ela recebe sempre que é chamada, para conseguir trabalhar.


**5. Coisas do dia a dia:**
Quando você chega perto de um NPC num jogo e aperta "E" para interagir com ele, o que o código do jogo faz por trás dos panos?


**6. Verdadeiro ou Falso:**
Na hora de dar nome para a função, a regra é que "o nome precisa deixar bem claro o que ela faz".
- [ ] Verdadeiro
- [ ] Falso


**7. Sobre o "Retorno" de uma função, qual é a opção certa?**
- [ ] A) É obrigatório que toda função mostre um resultado na tela.
- [ ] B) É o resultado que a função devolve para quem chamou ela no código.
- [ ] C) É a mensagem de erro que aparece quando a função dá problema.
- [ ] D) Funções com retorno são chamadas de "funções void".

### Nível Médio


**8. Funções com retorno vs. Sem retorno (Void):**
O texto dá dois exemplos do mundo real para explicar isso. Qual é a relação certa?
- [ ] A) Fechar a porta = Sem retorno | Comprar online = Com retorno
- [ ] B) Comprar online = Sem retorno | Fechar a porta = Com retorno
- [ ] C) Máquina da fábrica = Sem retorno | NPC do jogo = Com retorno
- [ ] D) NPC do jogo = Sem retorno | Máquina da fábrica = Com retorno


**9. Usando o exemplo do texto, explique com suas próprias palavras como funciona uma função sem retorno (void).**


**10. O que são "parâmetros opcionais"?**
- [ ] A) São funções que você escolhe se quer rodar ou não.
- [ ] B) São parâmetros que já vêm com um valor padrão; se a pessoa esquecer de preencher, o sistema usa esse valor para não quebrar o programa.
- [ ] C) São parâmetros que apagam os dados do usuário se ficarem em branco.
- [ ] D) São variáveis que só funcionam no Python.


**11. O exemplo do café (Parâmetros opcionais):**
Imagina que o cliente pede um café e esquece de falar o tamanho. Se o seu código **não** tiver um parâmetro opcional (ou padrão) para isso, o que deve acontecer com o programa?


**12. Organizando o código:**
Por que é melhor dividir um código gigante em várias funçõezinhas menores em vez de fazer tudo num blocão só?


**13. Qual dessas frases sobre funções está ERRADA?**
- [ ] A) O jeito de criar uma função muda dependendo da linguagem de programação.
- [ ] B) A palavra "def" é muito usada no Python para criar funções.
- [ ] C) Depois de criada, você pode chamar a função quantas vezes quiser.
- [ ] D) Uma função só pode receber um único parâmetro por vez, não importa a linguagem.


**14. Uma função pode chamar outra função dentro dela mesma para ajudar em alguma tarefa?**
- [ ] Sim, isso é super comum e ajuda bastante.
- [ ] Não, isso dá erro e trava o sistema.

### Nível Difícil


**15. O que é uma "Função Recursiva"?**
- [ ] A) É uma função que só pode ser lida de trás pra frente.
- [ ] B) É uma função feita para apagar arquivos.
- [ ] C) É uma função que chama a si mesma dentro do próprio código dela.
- [ ] D) É uma função que traduz o código para outra linguagem.


**16. O exemplo da recursividade:**
Qual objeto o texto usa para explicar como uma função recursiva funciona na prática?
- [ ] A) Uma porta fechando.
- [ ] B) Uma fábrica de máquinas.
- [ ] C) Um copo de café médio.
- [ ] D) Bonecas russas (aquelas que uma sai de dentro da outra).


**17. O perigo do "Loop Infinito":**
O texto avisa que funções recursivas precisam de um "ponto de parada". Se o programador esquecer de colocar isso, o que acontece com o código ou com o computador?


**18. O que são "Funções Lambda"?**
- [ ] A) São funções exclusivas de bancos de dados.
- [ ] B) São funções gigantescas, com milhares de linhas.
- [ ] C) São funções "anônimas" feitas para facilitar a vida e não ter que criar um bloco inteiro, usadas muito em códigos curtos de uma linha só.
- [ ] D) São códigos que rodam para sempre.


**19. Pensando em deixar o código fácil de ler, por que seria uma péssima ideia tentar fazer um programa complexo inteiro usando SÓ Funções Lambda (de uma linha)?**


**20. Desafio final:**
Imagina que você tem que criar uma função para **"Calcular a Média de um Aluno"** e dizer se ele passou de ano ou reprovou. Escreva aqui embaixo:
1. Um nome que faça sentido para essa função.
2. Quais **parâmetros** (dados) ela precisaria receber para fazer a conta.
3. Se ela seria uma função **Void (sem retorno) ou Com Retorno** (e qual seria esse retorno).
